## Overview

This prize funds a production-grade Ethereum–LEZ atomic swap protocol. A proof-of-concept implementation using Hash Time-Locked Contracts (HTLCs) exists at [eth-lez-atomic-swaps](https://github.com/logos-blockchain/eth-lez-atomic-swaps). The goal here is a robust, well-tested, and documented swap that can serve as a stable foundation for the Logos ecosystem — not a proof of concept.

## Motivation

Ethereum is the most liquid smart-contract chain. A trustless ETH↔LEZ swap without custodians, bridges, or wrapped tokens is a direct on-ramp between the two ecosystems and a prerequisite for meaningful cross-chain liquidity.

The existing proof-of-concept implementation demonstrates feasibility but is not production-ready: it lacks comprehensive testing, formal documentation of the atomicity argument, and handling of edge cases such as partial failures, timelock expiry, and concurrent swaps. This prize funds the work to close that gap.

Applicants may also explore alternatives to the HTLC approach — for example, adaptor signatures over secp256k1, which both Ethereum and LEZ support. Adaptor signatures avoid publishing the hash preimage on-chain, removing the correlation between the two swap legs visible to chain observers. This is not required, but submissions that improve on the HTLC privacy properties will be viewed favourably.

## Goals

**Trustless exchange**: ETH and LEZ tokens exchange hands across chains without any trusted third party. Either both legs complete or both refund — no state where one party receives funds and the other does not.

**Timelock refund**: Both legs support refund after timelock expiry. Note: LEZ does not yet have on-chain block time, so the LEZ-side timelock is enforced client-side, consistent with the approach in the reference implementation.

**Tested and documented**: The implementation covers edge cases, includes a formal atomicity argument, and is usable by third parties without reverse-engineering the code.

## Success Criteria

- [ ] A trustless ETH↔LEZ swap can be completed by two parties following the documented protocol.
- [ ] The two legs are atomic: no state exists where one party receives funds and the other does not.
- [ ] Both legs support timelock-based refund. The LEZ-side timelock is enforced client-side.
- [ ] The Ethereum-side contract is deployed on Ethereum Sepolia testnet and the LEZ program on LEZ testnet, both with verified identifiers.
- [ ] At least 5 complete ETH↔LEZ swaps are executed on testnet, involving at least 3 distinct counterparty pairs outside the submitting team.
- [ ] A coordination CLI or script guides both parties through the full swap lifecycle: setup, locking, claiming, and refund.
- [ ] A test suite covers: valid swap completion, timelock expiry and refund on both sides, and attempted double-claim.
- [ ] Documentation covers: protocol design, atomicity argument, timelock parameter rationale, security assumptions, and known limitations.

## Scope

### In Scope

- Ethereum-side swap contract (Solidity), deployed on Sepolia testnet.
- LEZ escrow program (Rust, Risc0) with conditional fund release and timelock-based refund, deployed on LEZ testnet.
- Cross-chain coordination CLI or script guiding both parties through the swap.
- Test suite covering the full swap lifecycle and failure cases.
- Documentation: protocol design, atomicity argument, security assumptions, known limitations, integration guide.

### Out of Scope

- ERC-20 token swaps — ETH and LEZ native tokens only.
- Multi-hop or routed swaps.
- A polished consumer UI — a working CLI is sufficient.
- Ongoing maintenance or security audits beyond initial delivery.
